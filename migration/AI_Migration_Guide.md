# 数字相移器 (100Hz - 10kHz) AI 迁移指南

## 1. 项目需求分析
* **输入**: 100Hz ~ 10kHz 连续单端正弦波。
* **输出**: 严格锁相同频正弦波，且支持与输入相位的 5° 步进调节。
* **痛点**: 100Hz 周期长达 10ms，传统全周期 ADC 采样延迟大；10kHz 要求较快的响应。

## 2. 核心架构设计 (大道至简)

根据我们在前置项目中验证的 **NCO Slaving (DDS底层相位直接操作)** 理论，数字相移器最稳定、最低延迟的架构如下：

### 方案优选：硬件输入捕获 + DDS 绝对相位注入 (无需独立软件PI环路)
1. **输入调理**: 将输入正弦波通过硬件比较器（或运放过零比较电路）转换为方波。
2. **频率与零相位获取**: 
   * 将方波接入 STM32 的 **定时器输入捕获 (Input Capture)** 引脚。
   * **频率获取**: 每次上升沿捕获，两次捕获的时间差 `\Delta T` 即为周期，直接算出高精度频率 `f = 1 / \Delta T`。
   * **绝对零相位基准**: 捕获中断触发的瞬间，**在物理时间上就是输入正弦波的 0 度过零点**！
3. **输出生成 (DDS + DAC)**:
   * **频段对齐**: 根据测得的 `f`，计算 DDS 的频率控制字 `FTW` 并写入。
   * **绝对相位注入**: 在输入捕获的中断处理函数中，**直接强制重置 DDS 的相位累加器**。
   * 公式：`dds_phase_acc = 用户设定的相移度数对应的累加值`。
   * **效果**: 每次输入波形过零，DAC输出的相位就被强制对齐一次。因为输出频率 FTW 算得很准，在两个捕获周期之间，输出波形不会偏离。这种“齿轮式”的同步可以做到示波器上**绝对纹丝不动**。

### 替代方案：ADC 全数字软件锁相环 (Software DPLL)
* 如果输入信号有极强噪声导致过零比较器频繁误触发，则使用 ADC + 软件锁相。
* 做法：降低采样率（如 50kHz），寻找波形峰值或过零点，计算误差送入 `dpll_core.c` 的 PI 控制器。

## 3. 迁移代码说明

本目录提取了前置项目中被证明极其稳定高效的代码：
* `dds_core.c / .h`: 纯净版的 DDS 驱动，包含了底层高速双缓冲 DMA 回调，以及我们发明的**强制相位注入接口**。
* `dpll_core.c / .h`: 纯净版的软件锁相环，剥离了具体的业务逻辑，保留了核心的 PI 积分控制、抗饱和、相位回绕计算。如果是复杂噪声环境，可用此模块。

## 4. 给 AI 助手的开发提示 (AI Coding Prompt)
如果你（AI）正在基于这个架构进行开发，请遵循以下步骤：
1. 配置一个硬件 Timer 用于 Input Capture（双边沿或单边沿均可，看具体硬件）。
2. 配置 DAC + DMA (Circular 模式)，使用 `dds_core.c` 里的缓冲逻辑。
3. 在 Input Capture 中断回调 `HAL_TIM_IC_CaptureCallback` 中：
   * 计算周期，得出 `freq`。
   * 调用 `DDS_Update_FTW( freq_to_ftw )`。
   * 调用 `DDS_Force_Phase( user_phase_degree )` 完成相移。
4. **不要在此处加入任何 printf 阻塞逻辑！**
