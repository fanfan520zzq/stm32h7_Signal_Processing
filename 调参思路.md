# 2023H DPLL 调参思路 (PI 控制器)

我们目前在 `2023h_dpll.c` 中使用的数字锁相环（DPLL）是一个非常经典的二阶锁相环结构。它直接控制 DDS 的**输出频率 (FTW)**，从而实现相位的锁定。

主要调参对象只有这两个：
```c
#define DPLL_KP 0.5f
#define DPLL_KI 0.05f
```

## 1. 核心运行逻辑

你的 DPLL 每次循环（大约 10 毫秒一次）会执行以下计算：

1. **计算相位差 (Error)**: 
   `error = 输入测得绝对相位 - 本地 NCO 预测相位`
   *(单位是弧度 rad，范围被折叠在 -π 到 π 之间)*
2. **积分项累加 (Integral)**: 
   `integral += KI * error`
3. **计算控制量 (输出频率)**: 
   `ftw_freq = 基准频率 + KP * error + integral`

**物理意义极其明确：**
* **KP (比例项)**：直接将当前的“相位差”转化为“瞬时频率的补偿”。如果 DDS 落后了，立马稍微加快频率追上去。
* **KI (积分项)**：它代表了“实际频率”和“基准频率”之间的**真实频差**。如果外部信号其实是 20001 Hz，而基准是 20000 Hz，积分项最终会稳稳地停在 `1.0` 左右。

---

## 2. KP 怎么调（管反应速度、抗抖动）

`KP` 决定了系统遇到相位偏差时，下意识“踩油门”的猛烈程度。
当前值 `KP = 0.5f` 意味着：如果相位差了 1 弧度（约 57 度），DDS 频率会瞬间增加 0.5 Hz 来追赶。

### KP 太大（例如 KP = 10.0 ~ 50.0）
* **现象**：
  * 系统极其神经质，稍微有一点测量噪声，输出频率就会剧烈抖动。
  * 示波器上的波形会像弹簧一样左右疯狂抽搐（高频抖动）。
  * 严重时会导致系统发散，永远锁不住。
* **处理**：降低 KP（例如从 10 降到 2，再降到 0.5）。

### KP 太小（例如 KP = 0.01）
* **现象**：
  * 系统像一辆动力不足的重卡。
  * 锁定的过程像乌龟一样慢，波形在示波器上缓缓移动，过了好几秒才对齐。
  * 遇到突发的相位跳变，它很久都追不回来。
* **处理**：增大 KP（例如从 0.01 提到 0.1，再提到 0.5）。

---

## 3. KI 怎么调（管消除静差、跟踪真实频率）

`KI` 决定了系统“学习并记住”真实频率偏差的速度。

### KI 太大（例如 KI = 2.0 ~ 5.0）
* **现象**：
  * 积分积累过快，导致超调严重！
  * 波形会在锁定的目标点附近，像钟摆一样**来回缓慢摇摆**（低频震荡）。
  * 即使基准频率和输入频率完全一致，波形也永远停不下来。
* **处理**：大幅降低 KI（通常 KI 应该是 KP 的 1/10 到 1/100，例如降到 0.05 甚至 0.01）。

### KI 太小（例如 KI = 0.0001）
* **现象**：
  * 波形锁得很快（因为 KP 在起作用），但总是**差一点点对不齐**。
  * 如果外部输入发生长期频率漂移（比如从 20000 漂到了 20005 Hz），波形会产生一个固定的、极其明显的相位差（也就是著名的“稳态误差”）。
* **处理**：适度增大 KI（例如提升到 0.01 或 0.05）。

---

## 4. 终极调参口诀与实战顺序

**调参第一定律：永远先调 KP，再调 KI。调 KP 时 KI 清零！**

### 第一步：关闭 KI（设为 0）
`KP = 0.5, KI = 0`
此时系统只有 P 控制。你调整信号源的相位，观察波形：
* 如果波形“能快速跟上，但不左右抽搐”，说明 KP 合适。
* 如果抖得厉害，减小 KP；如果跟得太慢，增大 KP。
*(对于当前系统，0.5 ~ 2.0 都是比较合理的 KP 范围)*

### 第二步：慢慢加入 KI
当 KP 稳定后，由于没有 KI，如果输入频率有一点点偏差，波形会存在固定相位差（无法完全重合）。
此时慢慢加大 KI：
`KI = 0.01 -> 0.05 -> 0.1`
* 加到刚好能够在中等时间（比如 0.5 秒内）让波形完全对齐重合即可。
* 如果加完 KI 后，波形开始像荡秋千一样来回摆，说明 KI 给大了，往回调小一半。

### 第三步：噪声抗性妥协
如果你的输入信号本身信噪比很差（测量出来的相位在跳），你**必须**把 KP 和 KI 都适当调小（例如 `KP=0.2, KI=0.01`），牺牲锁相速度来换取系统的平滑性，这相当于软件里的低通滤波。

## 5. 系统级避坑指南 (防死锁回顾)

1. **DMA 中断饿死**：DAC 的 1MHz 中断极快，绝不能在里面放任何浮点数或 `sin()` 函数。必须像现在的 `DDS.c` 一样，在主循环提前算好 `ActiveLUT`，中断里只做 `Buffer[i] = ActiveLUT[idx]` 纯内存拷贝。
2. **总线冲突 (OVR)**：ADC(2.4M) 和 DAC(1M) 同开 DMA 时容易导致 ADC Overrun。一旦 OVR，HAL 库会罢工死锁。必须在 `ADCTask.c` 里加 10ms 超时打断，并在启动采样前强行复位 `hadc->State = HAL_ADC_STATE_READY`。
3. **采样率崩溃**：重新生成 CubeMX 代码时，如果没同步 TIM4 参数，会导致采样率降到 10kHz，引起信号检测幅度全为 0 的惨剧。请时刻保证 TIM4 时钟分频和周期为 9。
